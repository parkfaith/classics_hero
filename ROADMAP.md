# 🗺️ Roadmap - 향후 개선 계획

## 현재 인프라 구성

### 기술 스택
- **Frontend**: Vercel (무료 티어)
- **Backend**: Render.com (무료 티어 - 콜드 스타트 15분)
- **Database**: Turso (무료 티어 - SQLite 호환)
- **AI**: OpenAI gpt-4o-mini
- **PWA**: 모바일 홈화면 추가 지원

### 현재 기능
- ✅ 6권의 고전 영어 책
- ✅ 6명의 역사적 인물과 대화
- ✅ 9개의 롤플레잉 시나리오 (3-5분)
- ✅ 읽기 모드 (번역, TTS)
- ✅ 말하기 모드 (발음 분석)
- ✅ 중요 단어/숙어 자동 추출 및 DB 캐싱
- ✅ PWA 지원 (오프라인, 홈화면 추가)

## 🎯 우선순위별 개선 계획

### Tier 1: 즉시 가능 (무료, 큰 효과) ⚡

#### 1.1 학습 진행 추적 시스템
**목표**: 사용자의 학습 동기 부여 및 재방문율 향상

**기능**
- 완료한 챕터/시나리오 체크 표시
- 독서 진행률 시각화 (진행 바, 퍼센트)
- 배지/업적 시스템
  - 첫 챕터 완료
  - 10개 챕터 완료
  - 모든 영웅과 대화
  - 첫 시나리오 완료
  - 연속 7일 학습
- 학습 통계 대시보드
  - 총 학습 시간
  - 읽은 단어 수
  - 완료한 시나리오 수

**구현 방법**
- localStorage 활용 (서버 비용 無)
- 간단한 JSON 구조로 진행률 저장
- 기존 progress 시스템 확장

**예상 효과**
- 재방문율 30-50% 향상
- 완료율 20-30% 향상
- 사용자 만족도 증가

---

#### 1.2 나만의 단어장 기능
**목표**: 학습한 단어 복습 및 장기 기억 강화

**기능**
- 읽기/대화 중 단어 저장 (별표 버튼)
- 저장된 단어 목록 페이지
- 단어 검색 및 필터링 (난이도, 날짜, 책/영웅별)
- 복습 모드
  - 플래시카드 형식
  - 랜덤 순서
  - "암기함/모름" 분류
- 단어 내보내기 (CSV, JSON)
- 단어 통계 (총 저장 단어, 복습 횟수)

**구현 방법**
- localStorage 활용
- 기존 Dictionary 컴포넌트 확장
- 새로운 "My Words" 페이지 추가

**예상 효과**
- 학습 효과 2-3배 향상
- 실용적인 기능으로 사용자 만족도 증가

#### 1.3 발음 점수 기록 및 히스토리

**기능**

**예상 효과**

- 학습 성취감 증가

### Tier 2: 단기 개선 (비용 소폭 증가) 💰

#### 2.1 Render 콜드 스타트 해결
**문제**
- 15분 비활성 시 서버 슬립
- 첫 요청 시 30초-1분 대기
- 사용자 경험 저하

**해결 방법**

**옵션 1: GitHub Actions Health Check (무료)** ⭐ 추천
```yaml
# .github/workflows/keep-alive.yml
name: Keep Backend Alive
on:
  schedule:
    - cron: '*/5 * * * *'  # 5분마다 실행
jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: Ping Backend
        run: curl https://classics-hero-api.onrender.com/api/health
```
- 장점: 무료, 설정 간단
- 단점: 5분 간격 한계

**옵션 2: Render 유료 플랜**
- $7/월 (Starter 플랜)
- 콜드 스타트 없음
- 더 나은 성능

**옵션 3: 다른 호스팅 이전**
- Railway.app (더 나은 무료 티어)
- Fly.io (더 빠른 콜드 스타트)
- Vercel Serverless Functions (프론트엔드 통합)

**추천**: GitHub Actions (무료, 즉시 적용 가능)

---

#### 2.2 AI 비용 최적화
**목표**: OpenAI API 비용 절감

**최적화 방법**
1. **대화 히스토리 요약**
   - 긴 대화 (10+ 메시지) → 요약본 생성
   - 토큰 사용량 50-70% 절감

2. **캐싱 강화**
   - 현재: vocabulary만 캐싱
   - 추가: 자주 묻는 질문 캐싱 (FAQ)
   - 영웅의 첫 인사 메시지 캐싱

3. **모델 선택 최적화**
   - 간단한 작업: gpt-3.5-turbo (더 저렴)
   - 복잡한 작업: gpt-4o-mini (현재)

4. **요청 최적화**
   - max_tokens 적절히 제한
   - temperature 조정으로 정확도 향상

**예상 효과**
- API 비용 30-50% 절감

---

#### 2.3 성능 모니터링 & 에러 추적
**목표**: 서비스 안정성 향상 및 문제 조기 발견

**도구**
1. **Sentry** (무료 티어)
   - 런타임 에러 자동 추적
   - 스택 트레이스 확인
   - 에러 알림 (이메일, Slack)

2. **Google Analytics 4**
   - 사용자 행동 분석
   - 페이지뷰, 체류 시간
   - 이탈률, 전환율

3. **Vercel Analytics**
   - 성능 모니터링 (Core Web Vitals)
   - 로딩 속도 추적

**예상 효과**
- 버그 발견 시간 70-80% 단축
- 데이터 기반 의사결정 가능

---

#### 2.4 개인화된 학습
**목표**: 사용자 레벨에 맞는 맞춤 학습

**기능**
1. **레벨 테스트** (첫 방문 시)
   - 간단한 발음 테스트 (5문장)
   - 독해 퀴즈 (3-5문제)
   - Easy/Medium/Advanced 자동 분류

2. **맞춤 추천**
   - 레벨에 맞는 책 우선 표시
   - 레벨에 맞는 영웅 추천
   - 적절한 난이도의 시나리오

3. **약점 분석**
   - 자주 틀리는 발음 패턴 분석
   - 어려워하는 어휘 유형 파악
   - 맞춤 연습 문제 생성

**구현 방법**
- localStorage에 사용자 레벨 저장
- AI로 약점 분석 (주 1회)
- 추천 알고리즘 구현

**예상 효과**
- 학습 효율 2-3배 향상
- 사용자 만족도 증가

### Tier 3: 중장기 개선 (유료 전환 필요) 🚀

#### 3.1 사용자 계정 시스템
**목표**: 다중 기기 학습 및 데이터 클라우드 동기화

**기능**
- Firebase Auth (Google, 이메일 로그인)
- 진행률 클라우드 동기화
- 다중 기기 학습 이어하기
- 사용자 프로필 설정
  - 학습 목표 (일일/주간)
  - 선호 난이도
  - 알림 설정

**비용**
- Firebase: 무료 티어 시작 (10만 사용자까지)
- 백엔드 DB 확장 필요

**예상 효과**
- 사용자 리텐션 50% 이상 향상
- 프리미엄 기능 기반 확보

---

#### 3.2 실시간 문법 교정
**목표**: 대화 중 즉각적인 피드백

**기능**
- 대화 중 문법 오류 자동 감지
- 즉각적인 수정 제안
- 문법 패턴 학습 (반복 실수 추적)
- 문법 팁 제공

**구현 방법**
- GPT-4 활용 (더 정확한 문법 분석)
- 스트리밍 응답으로 빠른 피드백

**비용**
- GPT-4 API 비용 증가 (2-3배)

**예상 효과**
- 영어 실력 향상 속도 3-4배
- 차별화된 프리미엄 기능

---

#### 3.3 소셜 기능
**목표**: 커뮤니티 형성 및 참여도 증가

**기능**
1. **친구 시스템**
   - 친구 추가/초대
   - 진행률 공유
   - 함께 읽기 챌린지

2. **학습 그룹**
   - 그룹 생성/가입
   - 그룹 챌린지 (주간 목표)
   - 그룹 리더보드

3. **리더보드**
   - 주간/월간 학습량 순위
   - 카테고리별 순위 (발음, 독서량, 시나리오)
   - 배지 획득 순위

**비용**
- 실시간 DB 필요 (Firebase Realtime Database)
- 서버 부하 증가

**예상 효과**
- 바이럴 효과 (친구 초대)
- 사용자 참여도 2-3배 증가

---

#### 3.4 오프라인 모드 강화
**목표**: 완전한 오프라인 학습 지원

**기능**
- 책 전체 다운로드 (Service Worker)
- 오프라인 단어장
- 오프라인 발음 연습 (녹음만)
- 재접속 시 자동 동기화

**구현 방법**
- Service Worker 고도화
- IndexedDB 활용
- 백그라운드 동기화 API

**예상 효과**
- 지하철, 비행기에서도 학습 가능
- 데이터 사용량 걱정 없음
